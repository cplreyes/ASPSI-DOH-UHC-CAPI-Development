## Identifier scheme

All identifiers are managed centrally so that records from any module can be joined for analysis.

| ID | Format | Generated by | Used in |
|---|---|---|---|
| `FACILITY_ID` | 8-digit numeric, derived from NHFR | Pre-loaded master | F0, F1, F3a, F3b, F4a (cross-ref) |
| `ENUMERATOR_CODE` | 3-digit numeric | Issued at training | All modules; auto-stamped per case |
| `SUPERVISOR_CODE` | 3-digit numeric | Issued at training | F0; cross-stamped on enumerator cases under supervisor's coverage |
| `LISTING_DATE` | YYYYMMDD | CSPro `systemdate(11)` | F3a, F4a |
| `VISIT_ID` | F0a key, composite | F0a | F0a, F0c |
| `PATIENT_LISTING_ID` | F3a key, sequential within facility | F3a | F3a в†’ F3b prefilled |
| `HOUSEHOLD_LISTING_ID` | F4a key, sequential within barangay | F4a | F4a в†’ F4b prefilled |
| `HCW_RESPONSE_ID` | F2 backend, UUID | F2 PWA | F2 staging DB; joined to F0b master via FACILITY_ID + role |
| `REPLACEMENT_FOR` | Original case ID | F0d | All replacement cases |
| `CAPI_VERSION` | `<MODULE>_v<MAJOR>.<MINOR>.<PATCH>` | Build process | Every case |

## Listing-to-interview launch pattern

For F3 and F4, the listing app launches the interview app via the CSPro `execpff()` function and prefills identifiers via PFF parameters:

```cspro
PROC START_INTERVIEW
  string interviewPffPath = "../F3b/F3b_PatientInterview.pff";
  setparam("Key", maketext("%v", PATIENT_LISTING_ID));
  setparam("INTERVIEWER_CODE", maketext("%v", ENUMERATOR_CODE));
  setparam("FACILITY_ID", maketext("%v", FACILITY_ID));
  execpff(interviewPffPath, stop);
end;
```

In the interview application's preproc:

```cspro
if sysparm("INTERVIEWER_CODE") <> "" then
  ENUMERATOR_CODE = tonumber(sysparm("INTERVIEWER_CODE"));
  protect(ENUMERATOR_CODE, true);
endif;
```

The interview application's identifiers (`PATIENT_LISTING_ID`, `FACILITY_ID`, `ENUMERATOR_CODE`) are auto-filled from the PFF and protected against editing.

## Source-of-truth rules

- **F1 DCF generator** (Python) is the source of truth for F1's data dictionary. Hand-edits to the .dcf file in CSPro Designer are not permitted; any change is made by patching `generate_dcf.py` and regenerating.
- **PSGC value sets** are self-served from the PSA 1Q 2026 publication; no ASPSI dependency.
- **HCW master roster** (F0b) is the canonical denominator for F2 monitoring; F2 admin portal reads from it.
