# F2 PWA Implementation Plan — M1: Generator v1 + Single-Section Render

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a TypeScript generator that parses `spec/F2-Spec.md` into typed Items + Zod schemas, then render F2 Section A (Healthcare Worker Profile) end-to-end in the PWA using react-hook-form + zodResolver — entirely from generated code.

**Architecture:** `scripts/generate.ts` (Node, invoked via `tsx`) parses the spec markdown, writes `src/generated/items.ts` + `src/generated/schema.ts`. The app imports both, composes them with react-hook-form, and renders Section A via a new `<Section>` + `<Question>` component pair. Submit logs JSON to console (no backend yet — M4).

**Tech Stack additions on top of M0:** react-hook-form 7, @hookform/resolvers 3, zod 3, tsx 4 (dev dep for running TS scripts under Node). No new build-time config — vite + vitest already handle everything.

---

## Scope note

This plan covers **M1 only** (12–15h of the 150–200h total). It assumes M0 has shipped — directories, Tailwind, shadcn Button, vite-plugin-pwa, vitest + RTL, ESLint flat config, Prettier are all in place.

Where M1 deliberately stays small:

- **One section only** (Section A, Healthcare Worker Profile). Extending to all 35 sections is M6.
- **Supported item types:** `short-text`, `long-text`, `number`, `single`, `single + specify`. Unsupported (`date`, `multi`, `multi + specify`, `grid-single`, `grid-multi`, `section-break`, `×N` multi-field) are logged and skipped. Unblocks M6.
- **No skip logic.** Items with `required: conditional` are rendered as optional. Skip logic + conditional show/hide is M3.
- **No cross-field rules.** The 20 POST rules from F2-Cross-Field.md are M7.
- **No i18n.** Labels are inline English. Bilingual bundle emission is M9.
- **No IndexedDB.** Submit handler logs JSON to console. Persistence is M2.
- **No backend.** No HTTP calls. M4 adds that.
- **No routing.** `/` renders Section A directly. Multi-page app is deferred until M8 (enrollment).

**Next plan to draft after this ships:** `YYYY-MM-DD-implementation-plan-M2-autosave-indexeddb.md` (M2 — Autosave + IndexedDB via Dexie, 8–10h).

```ts
export interface RawSection {
  id: string;
  title: string;
  body: string;
}

const SECTION_HEADER = /^## Section ([A-Z0-9]+(?:[0-9])?) — (.+?)\s*$/gm;

export function splitSections(markdown: string): RawSection[] {
  const sections: RawSection[] = [];
  const matches = [...markdown.matchAll(SECTION_HEADER)];

  for (let i = 0; i < matches.length; i++) {
    const current = matches[i];
    const next = matches[i + 1];
    const startIdx = (current.index ?? 0) + current[0].length;
    const endIdx = next ? next.index ?? markdown.length : markdown.length;
    const body = markdown.slice(startIdx, endIdx).trim();

    sections.push({
      id: current[1],
      title: current[2].trim(),
      body,
    });
  }

  return sections;
}
```

Note the regex: `[A-Z0-9]+(?:[0-9])?` matches single-letter IDs like `A`, `B`, `F` and compound IDs like `E1`, `E2` (F2 spec uses those for the BUCAS/GAMOT gated sub-sections).

```ts
export type RowFields = Record<string, string>;

const TABLE_HEADER = /^\|(.+)\|\s*$/;
const ALIGNMENT_ROW = /^\|[\s\-|:]+\|\s*$/;

function splitCells(line: string): string[] {
  return line
    .replace(/^\||\|$/g, '')
    .split('|')
    .map((c) => c.trim());
}

export function parseTableRows(body: string): RowFields[] {
  const lines = body.split('\n');
  let headerIdx = -1;

  for (let i = 0; i < lines.length; i++) {
    const line = lines[i];
    const next = lines[i + 1];
    if (TABLE_HEADER.test(line) && next && ALIGNMENT_ROW.test(next)) {
      const firstCell = splitCells(line)[0];
      if (firstCell === 'pdf_q') {
        headerIdx = i;
        break;
      }
    }
  }

  if (headerIdx === -1) return [];

  const header = splitCells(lines[headerIdx]);
  const rows: RowFields[] = [];

  for (let i = headerIdx + 2; i < lines.length; i++) {
    const line = lines[i];
    if (!TABLE_HEADER.test(line)) break;
    const cells = splitCells(line);
    if (!cells[0] || !/^Q\d+/.test(cells[0])) continue;

    const row: RowFields = {};
    header.forEach((col, idx) => {
      row[normalizeHeader(col)] = cells[idx] ?? '';
    });
    rows.push(row);
  }

  return rows;
}

function normalizeHeader(header: string): string {
  if (header.startsWith('label')) return 'label';
  if (header.startsWith('choices')) return 'choices';
  return header;
}
```

```ts
import type { Item, ParseResult, Section, UnsupportedItem, ItemType, Choice } from './types';

const SUPPORTED_TYPES: Record<string, ItemType> = {
  'single': 'single',
  'single + specify': 'single',
  'short-text': 'short-text',
  'long-text': 'long-text',
  'number': 'number',
};

const CHOICE_SEP = /\s*·\s*/;

export interface NormalizeRowResult {
  item?: Item;
  unsupported?: UnsupportedItem;
}

export function normalizeRow(row: RowFields, section: string): NormalizeRowResult {
  const rawType = (row.type ?? '').trim();

  if (!(rawType in SUPPORTED_TYPES)) {
    return {
      unsupported: {
        id: row.pdf_q,
        section,
        rawType,
        reason: unsupportedReason(rawType),
      },
    };
  }

  const type = SUPPORTED_TYPES[rawType];
  const hasOtherSpecify = rawType.includes('+ specify');
  const required = (row.required ?? '').trim() === 'Y';

  const { help, choicesText, min, max } = parseChoicesColumn(row.choices ?? '', type);

  const item: Item = {
    id: row.pdf_q,
    section,
    type,
    required,
    label: row.label ?? '',
  };

  if (row.legacy_q && row.legacy_q !== '—' && row.legacy_q !== '') {
    item.legacyId = row.legacy_q;
  }
  if (help) item.help = help;
  if (min !== undefined) item.min = min;
  if (max !== undefined) item.max = max;

  if (type === 'single') {
    item.hasOtherSpecify = hasOtherSpecify;
    item.choices = parseChoiceList(choicesText, hasOtherSpecify);
  }

  return { item };
}

export function parseSpec(markdown: string): ParseResult {
  const sections: Section[] = [];
  const unsupported: UnsupportedItem[] = [];

  for (const raw of splitSections(markdown)) {
    const preamble = extractPreamble(raw.body);
    const rows = parseTableRows(raw.body);
    const items: Item[] = [];

    for (const row of rows) {
      const normalized = normalizeRow(row, raw.id);
      if (normalized.item) items.push(normalized.item);
      if (normalized.unsupported) unsupported.push(normalized.unsupported);
    }

    sections.push({
      id: raw.id,
      title: raw.title,
      ...(preamble ? { preamble } : {}),
      items,
    });
  }

  return { sections, unsupported };
}

function parseChoicesColumn(
  raw: string,
  type: ItemType,
): { help?: string; choicesText: string; min?: number; max?: number } {
  const trimmed = raw.trim();
  if (!trimmed || trimmed === '—') return { choicesText: '' };

  const semiIdx = trimmed.indexOf(';');
  const head = semiIdx >= 0 ? trimmed.slice(0, semiIdx).trim() : trimmed;
  const tail = semiIdx >= 0 ? trimmed.slice(semiIdx + 1).trim() : '';

  const result: { help?: string; choicesText: string; min?: number; max?: number } = {
    choicesText: head,
  };
  if (tail) result.help = tail;

  if (type === 'number') {
    const rangeMatch = head.match(/(\d+)\s*[–-]\s*(\d+)/);
    if (rangeMatch) {
      result.min = Number(rangeMatch[1]);
      result.max = Number(rangeMatch[2]);
    } else {
      const minMatch = head.match(/min\s+(\d+)/i);
      if (minMatch) result.min = Number(minMatch[1]);
      const maxMatch = head.match(/max\s+(\d+)/i);
      if (maxMatch) result.max = Number(maxMatch[1]);
    }
  }

  return result;
}

function parseChoiceList(text: string, hasOtherSpecify: boolean): Choice[] | undefined {
  if (!text) return undefined;
  return text.split(CHOICE_SEP).map((label) => {
    const trimmed = label.trim();
    const isOther = hasOtherSpecify && /other[s]?(?:,\s*specify|\s*\(specify\))/i.test(trimmed);
    const choice: Choice = { label: trimmed, value: trimmed };
    if (isOther) choice.isOtherSpecify = true;
    return choice;
  });
}

function extractPreamble(body: string): string | undefined {
  const match = body.match(/^> \*Preamble.*?:\*\s*"(.+?)"/ms);
  return match ? match[1].trim() : undefined;
}

function unsupportedReason(rawType: string): string {
  if (/×\d+/.test(rawType)) return 'multi-field (×N) — handled in M6';
  if (rawType.startsWith('multi')) return 'multi-choice — handled in M6';
  if (rawType === 'date') return 'date picker — handled in M6';
  if (rawType.startsWith('grid')) return 'grid item — handled in M6';
  if (rawType === 'section-break') return 'section break — rendered in M3';
  return `unsupported type "${rawType}" — handled in a later milestone`;
}
```

```ts
import type { Item, ParseResult, Section } from './types';

export function emitItems(result: ParseResult): string {
  const header = [
    '// AUTOGENERATED — do not edit by hand.',
    '// Regenerate via `npm run generate`.',
    "import type { Section } from '@/types/survey';",
    '',
  ].join('\n');

  const sectionConsts = result.sections.map(emitSectionConst).join('\n\n');
  const sectionsArrayName = result.sections.map((s) => `section${s.id}`).join(', ');
  const sectionsArray = `export const sections: Section[] = [${sectionsArrayName}];`;

  const unsupportedBlock =
    result.unsupported.length > 0
      ? [
          '',
          '// unsupported items (skipped):',
          ...result.unsupported.map(
            (u) => `// - ${u.section}.${u.id} (${u.rawType}): ${u.reason}`,
          ),
        ].join('\n')
      : '';

  return [header, sectionConsts, '', sectionsArray, unsupportedBlock, ''].join('\n');
}

function emitSectionConst(section: Section): string {
  const itemsLiteral = section.items.map(emitItem).join(',\n    ');
  const parts: string[] = [`  id: '${section.id}'`, `  title: ${quote(section.title)}`];
  if (section.preamble) parts.push(`  preamble: ${quote(section.preamble)}`);
  parts.push(`  items: [\n    ${itemsLiteral},\n  ]`);

  return `export const section${section.id}: Section = {\n${parts.join(',\n')},\n};`;
}

function emitItem(item: Item): string {
  const fields: string[] = [
    `id: '${item.id}'`,
    `section: '${item.section}'`,
    `type: '${item.type}'`,
    `required: ${item.required}`,
    `label: ${quote(item.label)}`,
  ];
  if (item.legacyId) fields.push(`legacyId: '${item.legacyId}'`);
  if (item.help) fields.push(`help: ${quote(item.help)}`);
  if (item.min !== undefined) fields.push(`min: ${item.min}`);
  if (item.max !== undefined) fields.push(`max: ${item.max}`);
  if (item.hasOtherSpecify) fields.push(`hasOtherSpecify: true`);
  if (item.choices) {
    const choicesLiteral = item.choices
      .map(
        (c) =>
          `{ label: ${quote(c.label)}, value: ${quote(c.value)}${c.isOtherSpecify ? ', isOtherSpecify: true' : ''} }`,
      )
      .join(', ');
    fields.push(`choices: [${choicesLiteral}]`);
  }
  return `{ ${fields.join(', ')} }`;
}

function quote(s: string): string {
  return `'${s.replace(/\\/g, '\\\\').replace(/'/g, "\\'")}'`;
}
```

```ts
import type { Item, ParseResult, Section } from './types';

export function emitSchema(result: ParseResult): string {
  const header = [
    '// AUTOGENERATED — do not edit by hand.',
    '// Regenerate via `npm run generate`.',
    "import { z } from 'zod';",
    '',
  ].join('\n');

  const body = result.sections.map(emitSectionSchema).join('\n\n');
  return [header, body, ''].join('\n');
}

function emitSectionSchema(section: Section): string {
  const fieldLines = section.items.flatMap(emitFieldEntries).map((l) => `  ${l}`);
  const objectLiteral = `z.object({\n${fieldLines.join('\n')}\n})`;
  const constName = `section${section.id}Schema`;
  const typeName = `Section${section.id}Values`;
  return [
    `export const ${constName} = ${objectLiteral};`,
    `export type ${typeName} = z.infer<typeof ${constName}>;`,
  ].join('\n');
}

function emitFieldEntries(item: Item): string[] {
  const primary = `${item.id}: ${fieldSchema(item)},`;
  if (item.hasOtherSpecify) {
    return [primary, `${item.id}_other: z.string().optional(),`];
  }
  return [primary];
}

function fieldSchema(item: Item): string {
  switch (item.type) {
    case 'short-text':
    case 'long-text':
      return item.required ? 'z.string().min(1)' : 'z.string().optional()';
    case 'number': {
      let expr = 'z.coerce.number()';
      if (item.min !== undefined) expr += `.min(${item.min})`;
      if (item.max !== undefined) expr += `.max(${item.max})`;
      if (!item.required) expr += '.optional()';
      return expr;
    }
    case 'single': {
      const values = (item.choices ?? []).map((c) => `'${c.value.replace(/'/g, "\\'")}'`);
      const base = `z.enum([${values.join(', ')}])`;
      return item.required ? base : `${base}.optional()`;
    }
  }
}
```

```tsx
import { useFormContext } from 'react-hook-form';

  const { register, watch } = useFormContext();
  const currentValue = watch(item.id);
  const showSpecify =
    item.hasOtherSpecify &&
    item.choices?.some((c) => c.isOtherSpecify && c.value === currentValue);

    </div>
  );
}

function renderControl(

          type="text"
          className="rounded border border-input bg-background px-3 py-2"
```

```tsx
import { zodResolver } from '@hookform/resolvers/zod';
import type { ZodTypeAny } from 'zod';
import type { Section as SectionModel } from '@/types/survey';
import { Button } from '@/components/ui/button';
import { Question } from './Question';

interface SectionProps<T extends Record<string, unknown>> {
  section: SectionModel;
  schema: ZodTypeAny;
  onSubmit: (values: T) => void;
}

export function Section<T extends Record<string, unknown>>({
  section,
  schema,
  onSubmit,
}: SectionProps<T>) {
  const methods = useForm<T>({
    resolver: zodResolver(schema),
    mode: 'onSubmit',
  });

  return (
    <FormProvider {...methods}>
      <form
        onSubmit={methods.handleSubmit((values) => onSubmit(values as T))}
        className="mx-auto flex max-w-xl flex-col gap-4 p-6"
        noValidate
      >
        <header className="flex flex-col gap-1">
          <h2 className="text-2xl font-semibold tracking-tight">
            Section {section.id} — {section.title}
          </h2>
          {section.preamble ? (
            <p className="text-sm text-muted-foreground">{section.preamble}</p>
          ) : null}
        </header>

        {section.items.map((item) => (
```

## Notes

- **Why split types between `scripts/lib/types.ts` and `src/types/survey.ts`?** The generator runs under its own tsconfig (`tsconfig.scripts.json`) and uses Node-only deps (`node:fs`, `node:path`). The app code runs under `tsconfig.app.json` with DOM types. Sharing a single type file across both would force the app tree to include Node types or vice versa. Two small identical files is cheaper than the cross-cutting cast needed to share one. A drift-check (snapshot test that `emitItems` output typechecks against `src/types/survey`) lands in M11 hardening.

- **Why no i18n yet?** M9 is the milestone that introduces `public/locales/en.json` + `fil.json` + `react-i18next`. For M1, inline English labels are plenty — the generator's output just embeds the verbatim spec label as a string. When M9 lands, the emitter gains a second pass that extracts labels into a locale bundle and emits `i18nKey: 'Q3.label'` on each Item.

- **Why log unsupported items instead of failing?** The generator will hit ~74 unsupported items across all 35 sections (date fields, multi-choice, grids, ×N multi-field). Failing fast would block M1 entirely. Logging preserves visibility while letting supported items (Section A's 9 items for this milestone) flow through. M6 expands the supported-type matrix; at that point the unsupported list should shrink toward zero.

- **Why `z.coerce.number()` instead of `z.number()`?** HTML number inputs submit strings. `z.coerce.number()` converts at validation time. This matches how react-hook-form + zodResolver expects to see raw DOM values.

- **Why no resume/draft yet?** A page reload wipes form state in M1. That's the precise bug M2 fixes by wiring RHF's `watch` → Dexie `put` to the `drafts` table. M1 intentionally does not try to preempt that work.
