## Issues

### ISSUE-001 [HIGH] Raw validation errors leaking to UI

**Category:** Functional / UX copy
**Pages:** Section A — every numeric field
**Repro:**
1. Enroll with HCW-QA-001 / Makati Facility, land on Section A
2. Click "Next section" without filling anything
3. Observe inline error messages

**Expected:** User-facing copy like "Please enter a number" or "This field is required"
**Actual:** `Expected number, received nan` appears under Q4 (age), Q9 (Years/Months), Q10 (days/week), Q11 (hours/day)

This is the default Zod error string and it leaks the schema's internals. Healthcare workers in the field will not know what "nan" means.

**Evidence:** `screenshots/next-without-required.png`

---

### ISSUE-002 [MEDIUM] Q6 and Q8 absent from Section A — verify against spec

**Category:** Functional / Spec alignment
**Pages:** Section A
**Repro:**
1. Enroll, land on Section A
2. Scroll the question list
3. Visible questions: Q1, Q2, Q3, Q4, Q5, **(Q6 missing)**, Q7, **(Q8 missing)**, Q9, Q10, Q11

I tested toggling Q7 = Yes and Q5 = "Other (specify)" — neither revealed Q6 or Q8. Either they are conditional on a different answer (Q3 sex? Q5 specific role?) or the question numbering jumps by spec.

**Action:** Cross-check against the source questionnaire spec at `1_Projects/ASPSI-DOH-CAPI-CSPro-Development/raw/` to confirm intent. If they are conditional, document the trigger. If they are spec gaps, the build is fine — just note for whoever reads the data later.

**Evidence:** `screenshots/section-a-loaded.png`, `screenshots/next-without-required.png`

### ISSUE-003 [MEDIUM] FIL language toggle has no effect on visible UI

**Category:** UX / i18n
**Pages:** All (header toggle)
**Repro:**
1. Click "FIL" in the header
2. FIL button shows pressed state (dark green)
3. All visible page text remains English

Per memory notes (`project_aspsi_translations_pipeline`), this is expected — Filipino strings have not been delivered yet, only the switcher infrastructure is in place. But there is no user-facing signal. A real enumerator will assume the button is broken.

**Recommendation:** Either disable the FIL button with a "Coming soon" tooltip, or render a subtle banner ("Translations pending — fallback to English") when FIL is pressed but i18n keys are missing.

**Evidence:** `screenshots/fil-locale.png` (notice all text identical to `screenshots/initial.png`)

### ISSUE-004 [LOW] Middle Initial flagged as Required

**Category:** Functional / Cultural fit
**Pages:** Section A, Q1
**Observation:** Middle Initial input shows a red "Required" marker on validation failure.

In Filipino name conventions middle initial is typically **optional** — many people use only first + last. Forcing it as required will frustrate enumerators or cause data quality workarounds (entering `-` or `X`).

**Recommendation:** Confirm against the spec; if optional, drop the required validator on this field.

**Evidence:** `screenshots/next-without-required.png`

---

### ISSUE-005 [LOW] Inconsistent label casing for "Other" options

**Category:** Content / Polish
**Pages:** Section A
**Observation:**
- Q2 employment radio uses `Other, specify` (comma form)
- Q5 role radio uses `Other (specify)` (parens form)

Both reveal a "Please specify" textbox correctly, so functionally fine — but inconsistent copy.

**Recommendation:** Pick one form and apply across all "Other" options in the spec.

**Evidence:** `screenshots/section-a-loaded.png` (Q2 vs Q5 labels)

---

### ISSUE-006 [LOW] Locked sidebar sections appear as plain buttons in a11y tree

**Category:** Accessibility
**Pages:** Sidebar (all 9 sections)
**Observation:** Sections B-9 are visually styled as locked (gray text, lock icon) and clicking them does not navigate (good — gating works). But in the accessibility tree they appear as enabled `[button]` with no `aria-disabled`. Screen reader users will believe they can navigate to all sections.

**Recommendation:** Add `aria-disabled="true"` and `aria-describedby="locked-reason"` (e.g. "Complete previous section to unlock") to locked section buttons.

**Evidence:** `screenshots/click-locked-section.png` (clicking @e5 stayed on Section A)

---

### ISSUE-007 [LOW] "Test Facility" present in dev seed list

**Category:** Content / Production hygiene
**Pages:** Enrollment screen
**Observation:** After clicking "Refresh facility list", the dropdown contains `Makati Facility` and `Test Facility`. The latter is clearly dev seed.

**Recommendation:** Verify the production build excludes seed data from `Test Facility`. Trace the data source — if it comes from a backend API, check that staging/prod backends don't return it. If it's a hardcoded fallback for offline mode, gate it behind a dev flag.

**Evidence:** `screenshots/refresh-result.png`

### ISSUE-008 [LOW] Mobile viewport — floating arrow indicator clips off-screen

**Category:** Visual

**Observation:** At 375×812 viewport (iPhone X / Android equivalent), a circular arrow indicator on the right edge of the form panel is half off-screen.

## What is working well
