# F2 PWA — Accessibility (WCAG 2.1 AA)

Reference for the F2 PWA (HCW survey + Admin Portal). Target: **WCAG 2.1 Level AA**.
A government health instrument has to be usable by enumerators and respondents
with low vision, motor, or cognitive constraints, on field tablets.

## Automated coverage (runs in CI)

| Gate | What it checks | Where |
|---|---|---|
| **axe-core component tests** | DOM-level a11y (roles, names, labels, ARIA, focus order) across 37 cases — chrome, survey, admin shell, admin dashboards/editors | `src/a11y.test.tsx`, `src/components/survey/survey.a11y.test.tsx`, `src/admin/admin.a11y.test.tsx`, `src/admin/admin.dashboards.a11y.test.tsx` (via `src/test/axe-helpers.ts`) |
| **contrast gate** | WCAG contrast ratios for every theme foreground/background pair, light + dark — **closes the gap** left by axe's disabled `color-contrast` rule (jsdom can't compute CSS) | `scripts/check-contrast.mjs` (runs in `npm run build`) |

The axe component tests deliberately disable `color-contrast` (no computed CSS in
jsdom) and `region` (component, not full-page) — that's why the standalone
contrast gate exists (it parses the real `index.css` tokens at build time), and
why full-page checks below are run in a real browser.

## Known contrast exceptions (pending design review)

The token-level gate surfaced pairs that **don't meet AA** today. These are
DOH-anchored Verde Manual colors — changing them is a design decision
(`DESIGN.md` Decisions Log + approval). The gate **pins each at a regression
floor** (can't get worse) rather than failing the build; they should be raised
to AA in a palette pass:

| Theme | Pair | Ratio | AA? | Where it shows |
|---|---|---|---|---|
| light | `warning` text on `background` | 2.70:1 | ✗ (need 4.5) | Lock-notification strip, broadcast banner, "warn"-severity Review alerts |
| light | text on `warning` fill | 2.70:1 | ✗ | Any paper-on-ochre warning chip |
| dark | `primary` button text | 2.99:1 | ✗ | Primary buttons in dark mode |
| dark | `destructive` button text | 3.13:1 | ✗ for text (meets 3:1 UI) | Destructive buttons in dark mode |

Suggested fix (for the design pass): darken the light **ochre** `--warning`
(`#C68A2E`) toward ~`#9A6A1E` for ≥4.5:1 on paper, and nudge the dark-mode
`--primary` / `--destructive` so their `*-foreground` text clears 4.5:1. All
other pairs (body, muted labels, links, primary/destructive in light) pass AA.

## Re-run

```sh
cd deliverables/F2/PWA/app

# Full-page Lighthouse a11y (HCW + admin login):
npm run audit:a11y                  # → lighthouse-a11y.html (HCW enrollment)
npx lighthouse https://f2-pwa-staging.pages.dev/admin/login \
  --only-categories=accessibility --chrome-flags="--headless=new" \
  --output=html --output-path=./lighthouse-a11y-admin.html --quiet

# Component + contrast gates (the CI ones):
node scripts/check-contrast.mjs    # WCAG contrast (also runs in npm run build)
npm test -- a11y                   # the 37 axe-core component tests
```

Re-run the Lighthouse baseline after any visual/layout change; re-check the
contrast gate after any `--*` token edit in `index.css`.
