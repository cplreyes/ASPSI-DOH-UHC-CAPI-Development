## Conventions

### Frontmatter
Every file has YAML frontmatter. Minimum fields by file type:
- **`product-backlog.md`**: project / client / implementer / data_programmer / qa_tester / contract / engagement_window / last_updated
- **`sprint-current.md`** + **`sprints/sprint-NNN.md`**: sprint / start / end / status / sprint_length / deliverable_anchor
- **`epics/epic-NN-*.md`**: epic / title / phase / status / last_updated
- **`standups/YYYY-MM-DD.md`**: date / project / sprint / sprint_day / sprint_status

### Task IDs

Format: `E{epic}-{track or instrument}-NNN`

Examples:
- `E0-010` — Epic 0 (Project Management), task 010
- `E2-F1-005` — Epic 2 (Questionnaire Design), F1 instrument, task 005
- `E3-F2-001` — Epic 3 (CAPI Application), F2 instrument, task 001
- `E4-CSWeb-003` — Epic 4 (Backend), CSWeb track, task 003

Each epic file has a `Task conventions` section that documents its specific track / instrument breakdown.

### Inline fields

Tasks use Obsidian-style inline fields for parser-readiness (Markdown is canonical; ClickUp is a downstream projection):

```
- [ ] **E0-010** Define weekly status update format `status::todo` `priority::high` `estimate::2h`
  - Optional sub-bullets for context, blockers, or done-summaries
```

Field values:
- `status::` — `todo` / `in-progress` / `done` / `blocked` / `ongoing` (recurring tasks)
- `priority::` — `critical` / `high` / `medium` / `low`
- `estimate::` — duration (`30m`, `2h`, `1d`) or `recurring` for ceremonies, or `TBD` if unknown

### Cross-references

Use Obsidian wikilinks. Common patterns:
- Epic → PB row: `[[../product-backlog#Epic 2 — Survey Questionnaire Design & Dictionary|PB Epic 2]]`
- Sprint → Epic task: `[[epics/epic-00-project-management#E0-010|E0-010]]`
- Either → CAPI workflow phase: `[[../../../../2_Areas/IT-Standards/templates/CAPI-Development-Workflow|CAPI Development Workflow]]`

### Done-task discipline

When a task closes, retain the `[x]` checkbox + a one-line done-summary as a sub-bullet. Don't delete completed tasks — they're the audit trail.

```
- [x] **E0-060** Attend Apr 13 LSS meeting `status::done` `priority::critical`
  - Done 2026-04-13. Notes ingested at wiki/sources/Source - ASPSI Team Meeting 2026-04-13.md.
```

## Sprint lifecycle

| Day | Activity |
|---|---|
| **Mon** | **Planning** — Review PB headline, pick committed items from epic files into `sprint-current.md`, set sprint goal, fill DoD checklist |
| **Tue–Thu** | **Execution** — Work the items. Daily standup files auto-generated. Update task status inline as work progresses |
| **Fri** | **Close** — Mark items done in epics, write retro notes in `sprint-current.md`, update PB headline + epic state cells, copy `sprint-current.md` → `sprints/sprint-NNN.md`, reset `sprint-current.md` for next week's planning |

## Automation hooks

- **SessionStart hook** auto-writes `standups/YYYY-MM-DD.md` per Claude session, pulling Stakeholder Context from PB and tactical state from `sprint-current.md`.
- **Friday retro seed** prepares the 4-question retro template in `sprint-current.md` Friday morning.
- **Daily standup poster routine** (`trig_0123wNRHkgP1p4eW3dxGrWU1`) posts a Slack digest to `#capi-scrum` weekday mornings via the Slack MCP.

See `2_Areas/IT-Standards/templates/CAPI-Development-Workflow.md` for the 12-phase workflow that the 13 epics map onto.
